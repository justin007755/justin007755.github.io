用Hexo在Github上搭建自己的博客-1

Jul 22, 2016 | 写作 | Hits

以前我的博客放在chinaunix和51CTO里，但一直对它们的页面排版和内容组织等等都不太满意，直到某一天浏览李笑来的博客，他提到了用Hexo在Github里搭建自己的博客比较好，所以花了一天的功夫实践了一下。

关于Hexo:

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

准备搭建：

主要参考的是李笑来的博客（见附录），但他是基于Macbook写的教程，而本人用的是比较low的win7，所以在实践的过程中不得不踩一些坑，也查了一些其它的资料（见附件），断断续续花了大约一天的时间，终于把一个缺省的静态页面部署到Github上了。

具体步骤：

1） 在Github上申请帐号，并创建一个Repo，名字格式为yourname.github.io。
用来做博客的远程创库、域名、服务器之类的。
2） 本地安装Git。
用来把本地的hexo内容提交到github上去。
这一步碰到一些问题。首先，应该如何选择Git的客户端？如何用Macbook，一条命令即可（brew install git），类似Linux的yum命令，但windows里比较少有类似的命令（有一个叫“chocolatey”的应用，但是可以用来下载安装的包不太全），所以只能单独去下安装包。其实Git的客户端比较多，网上用的比较多的是github自己的客户端以及git-for-windows。这两个我都下载用了一下，不得不说github是真爱，陪我走到了最后（其实也有很多人用的是git-for-windows）。所以在下面的截图里看到的都是Git Shell(Powershell)。其实它们的GUI界面也挺好用的，我试了一下Github的GUI，图标是蓝猫的那个。

https://desktop.github.com/

https://git-for-windows.github.io/

3） 本地安装npm。
Npm其实就是nodejs，用来生成静态页面的。一般来说，用nvm(Node Version Manager)来安装npm比较好，因为npm可能不止一个版本。用下面的链接可以下载一个nvm安装好，然后用一条命令（nvm install 4.4.5）去安装和管理npm。通常情况下，装完nvm以后，执行命令已经被添加到Path里了，所以可能在任何地方运行nvm。当然，一开始我也是下载了npm包手动装上的，貌似也没问题。

https://github.com/coreybutler/nvm-windows/releases
https://github.com/coreybutler/nvm-windows


4） 安装Hexo。
这一步花的时间最多，也可能是前面的步骤未作好导致的。
首先要注意的是，必须用Git shell去执行npm来安装hexo。另外，由于npm要去指定的地方下载hexo，所以务必保证下载的地址是可访问的。我碰到的一个问题是，在下载时光标一直在忙状态，但是就是没有进度，后来发现是公司的代理服务器把下载的地址给屏蔽了，用自己的手机做热点，问题马上解决。

4.1）执行“npm install hexo-cli -g”和 “npm install hexo save”，其中hexo应该是http服务器，hexo-cli是一个命令行工具，用来操作hexo服务器。

有一个告警（WARN），但应该可以忽略。

屏幕输出继续…..

屏幕输出继续…..

直到最后看到“INFO Start blogging with Hexo！”。

4.2） 初始化，执行“hexo init ”， 是你的本地文件夹，我的是“D:\Learning\hexo”, 也可以cd到目标目录，执行“hexo init“。有的文章说，至此全部安装工作就完成了，但我还是多做了一步，“cd ”然后“npm install”，这个是参考https://hexo.io/zh-cn/docs/setup.html的。

4.3）启动本地的Hexo Server：“hexo server”。

用“netstat -an”可以看到hexo在4000端口上监听：

用浏览器打开的缺省主页：

到这里，我们已经可以在本地打开一个缺省的主页了，和安装一个Apache的http server差不太多。

4.4）然后，如何把自己的站点建在Github上，并用git来从本地同步你的博客到Github上？关于这一点，李笑来的文章里谈到了如何实现，其实就是先在Github上建一个和你的Github用户名相同的Repo（见具体步骤的第一步），然后用git的clone命令把这个Repo拷贝到自己的本地工作目录。

cd D:\Learning\hexo_github
git clone https://github.com/yourname/yourname.github.io.git

然后，还是要初始化一下你自己的这个本地站点：

hexo init yourname.github.io
cd yourname.github.io
npm install hexo-deployer-git –save
hexo generate
hexo server


最后，就是修改一下_config.yml 配置文件的最后几行，在命令行里运行“hexo deploy”。

deploy:
  type: git
  repo: https://github.com/justin007755/justin007755.github.io.git

打开“yourname.github.io”这个网址即可以看到自己的主页了，我的是“https://justin007755.github.io/”

接下来还得学习一下如何如何更好地配置hexo，更好地利用它来组织自己的博客，慢慢弄吧….
