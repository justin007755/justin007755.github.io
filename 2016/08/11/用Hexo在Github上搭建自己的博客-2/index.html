首先在知乎上查了一下合适的主题（Theme）, Maupassant比较符合我的风格，简约而不简单，果断入手。
https://www.zhihu.com/question/24422335

安装Maupassant:

三条命令搞定，唯一碰到的问题是安装sass时报错，应该是被墙了，换了VPN之后搞定。

Install theme and renderers:
$ git clone https://github.com/tufu9441/maupassant-hexo.git themes/maupassant
$ npm install hexo-renderer-jade –save
$ npm install hexo-renderer-sass –save

安装完成后编辑Hexo目录下的 _config.yml，将theme的值改为maupassant。

然后用“hex new”命令生成新的markdown文件，用MarkdownPad编辑。

将独立域名与 GitHub Pages 的空间绑定：

还差一步，就是申请一个独立域名，并把它与自己的Github页面空间绑定起来。在网上查了一圈，有不少如何操作的文章（见附录）。总结下来，就如下两步：

a） 选购独立域名

网上大多数人都推荐去goDaddy买域名，但我在网上比较了一下，发现万网的域名比goDaddy上要实惠一些，而且万网实际上也在阿里云下，估计不会离谱到哪里去，于是果断在万网买了域名（justinzhang.me），可以用支付宝扫描支付，相当方便（阿里的收钱工具一向都做的很好用：））。

b）修改DNS

将万网的Nameservers更改成 f1g1ns1.dnspod.net 和 f1g1ns2.dnspod.net（DNSpod的默认NS记录），用dnspod来做域名解析:

用Markdown编辑文章：

前面的铺垫完成以后，终于可以把自己的博客（其实只是断断续续地写了几篇文章而已：（）从chinaunix移到自己的独立域名下了，空间当然还是用Github的场子。

之前都是用chinaunix的编辑器来编辑博文的，到了Github就入乡随俗，改用Markdown吧。在网上下了一个免费的markdown编辑器（MarkdownPad2），感觉还不错，试用了一部分Markdown的语法，基本是所见即所得。有两个地方值得注意：

a）博文所用的图片最好用一个云服务器来托管，否则占Github的空间太大，而且不好在markdown Pad里调试。我申请的是七牛的空间，有10G的免费容量，感觉服务还不错，不过需要实名验证。

用Markdown Pad编辑，采用七牛作图床，图片可以马上显示出来。

b）用Markdown画表格稍微有点麻烦，但是Markdown Pad 2自带了画表格的解析器（需要在左下角选Markdown (Extra)），如下。否则只能用html的标记来画表格了，如 < br >。

c）运行“Hexo d –g”
