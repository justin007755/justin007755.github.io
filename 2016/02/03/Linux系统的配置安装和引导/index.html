Linux系统的配置安装和引导

Feb 3, 2016 | IT技术 | Hits

从一开始研究Linux，就对系统的运行原理很感兴趣，其中最想搞清楚的莫过于系统是如何被配置和安装的，以及如何被启动起来的。关于前一个问题，个人觉得鸟哥的Linux里讲的还是比较清楚的，下面我们来一一分析一下。

系统是如何被配置和安装的？

Linux虽然是个软件操作系统，但是其载体以及操控的对象还是硬件，所以解决这个问题还是得从计算机硬件开始。由于通常情况下我们是将操作系统安装到硬盘上的，所以我们首先应该了解的是硬盘是如何被分区组织的。通常情况下我们的电脑可能不至一块硬盘，所以不同的硬盘在系统里有不同的设备文件名，比如IDE接口的硬盘如果其接在IDE1的扁平电缆上，则其设备名一般叫”/dev/hda”和”/dev/hdb”，如果其接在IDE2的扁平电缆上，则其设备名一般叫”/dev/hdc”和”/dev/hdd”。另一个重要的问题是这些硬盘如何被分区？在硬盘的第一个扇区有两个重要的东西如下：
主引导分区(Master Boot Record, MBR)：可以安装引导加载程序的地方，有446 bytes
分区表(partition table)：记录整块硬盘分区的状态，有64 bytes
其中分区表就是用来记录整个硬盘分区情况的，借用下面鸟哥的一张图来说明。所以：
1）所谓分区只是对那个64bytes的分区表进行设置。
2）默认分区表最多能写入4组分区信息（64bytes只能记录4条数据），称为主（Primary）或扩展（Extented）分区。
3）扩展分区最多只能一个，逻辑分区是由扩展分区持续切割出来的分区。
4）能够被格式化后做为数据访问的分区为主分区和逻辑分区，扩展分区的目的是使用额外的扇区来记录分区信息，本身无法被格式化，所以扩展分区只是一个记录区而已，真正被使用而且在df命令下能显示的只是主分区和逻辑分区。

系统的引导过程

引导（bootstrapping）是“启动计算机”的标准说法。在引导一台机器的时候，它是从执行保存在ROM中的代码开始的。在PC上，这种初始化的引导代码叫做BIOS。BIOS通常让用户选择从什么设备进行引导，一旦确定从什么设备引导系统，它将尝试加载设备上开头512个字节的信息，也即MBR（Master Boot Record）。MBR里有一个程序，它告诉计算机从哪个分区加载第二阶段的引导程序，即boot loader。之后，就由boot loader负责加载内核。

init是系统引导起来以后第一个运行的进程（Redhat7以后改用Systemd了），从很多方面它都是最重要的守护进程。init至少定义了7个运行级别，每一个都代表系统应该补充运行的一组特定服务：
1）运行级别0完全关闭系统
2）运行级别1或S代表单用户
3）运行级别2～5包含连网支持
4）运行级别6是Reboot
大多数系统正常的多用户运行级别是2或3，运行级别5常用于X Windows的登录进程。/etc/inittab文件告诉init在它的每个运行级别上要做什么事情。随着机器的引导，init从运行级别0开始，一级一级往上运行到/etc/inittab里所设置的默认运行级别。

#### /etc/inittab #####
# 0 - halt (Do NOT set initdefault to this)
# 1 - Single user mode
# 2 - Multiuser, without NFS (The same as 3, if you do not have networking)
# 3 - Full multiuser mode
# 4 - unused
# 5 - X11
# 6 - reboot (Do NOT set initdefault to this)
#
id:5:initdefault:
##################

启动脚本的主拷贝位于/etc/init.d这个目录下，但是当主脚本把系统引入到一个新的运行级别时，它不是直接在/etc/init.d下找，而是查找叫做rclevel.d的目录，比如rc0.d，然后这个目录下的脚本都是符号链接到init.d目录的脚本上。另外，Redhat还有一个在引导时运行的/etc/rc.local脚本，它作为启动过程的一部分是最后一个运行的脚本。
